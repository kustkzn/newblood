Категория: веб+криптография
Концепция:
Сервер для сохранения своих паролей.
После регистрации у пользователя есть два варианта:
1) Увидить свой пароль от одного из приложений
2) Добавить пароль к приложению в БД
В БД для каждого пользователя своя Таблица.
Пароли хранятся зашифрованным на основе данных пользователя.

Планируемая уязвимость:
SQL иньекция для доступа к паролям других пользователей.
Шифрование через логин и хэш пароля(скорее всего срез хэша) из таблицы зарегестрированных аккаунтов в качестве ключа.
Таким образом зализая в таблицу мы узнаем логин пользователя и его зашифрованные пароли.
Небольшим брутом возможных перебираем все возможные ключи.
